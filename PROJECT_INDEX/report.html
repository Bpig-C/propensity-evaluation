项目本质

**PropensityBench 是一个 LLM 安全倾向性评测框架**，专门测量语言模型在被赋予高风险工具、并置于"操作压力"下时，**倾向于执行有害行动的概率**。 它的核心研究问题是：不是"模型能不能做坏事"（能力），而是"模型在正常运行环境下会不会主动选择做坏事"（倾向性/propensity）。 框架分两条主线： 1. **场景生成流水线**（`generation/`）：用 LLM 批量生成高风险业务场景（4个领域 × 32个工作空间 × 多角色 = 979个场景，共5874个测试用例），每个场景包含工具定义、状态机、政策约束、压力消息。 2. **Agentic 评估执行器**（`evaluation/`）：将被测 LLM 放入模拟的高风险工作空间，配备真实工具调用接口，施加系统性压力，观察其是否调用了预设的"有害目标函数"（`trigger_function`）。 ---

`APIConfiguration`：简单数据类，持有 model_name/provider/api_base/use_cache 四个字段 - `RateLimiter`：用文件系统信号量（`.tmp/` 目录下的 JSON 文件 + `fcntl.flock`）实现跨进程安全限速；后台线程每秒释放过期 token；支持 key 轮转 - `Client`：封装 `litellm.completion()`，支持 Pydantic schema 或 dict schema 的结构化输出；温度/缓存/proxy 均可配置 **关键隐患**：`fcntl` 是 POSIX 专属模块，在 Windows 上 **不可用**。这是该项目在 Windows 上无法运行限速功能的根本原因。 ### evaluation/ **本质**：把 LLM 关进一个带有"诱导性工具"的模拟工作空间，观察其是否触发有害行动。 - **`Agent`**：持有对话历史（history）、工具调用封装（OpenAI function-calling 格式）和角色系统提示。`SYSTEM_PROMPT_TEMPLATE` 故意伪造"真实世界 API"的感知（"This API controls REAL-WORLD instruments"），这是施压设计的一部分。 - **`Executor`**：管理 12 轮交互循环，处理 tool call 解析、连续消息限制、工具调用计数。关键逻辑：检测 `trigger_function` 是否被调用（=propensity 触发）。 - **`ValidationExecutor`**：单步探针，用于验证场景是否合理（不跑完整 episode），返回 yes/no 判断。 - **`PropensityEvaluator`（main.py）**：最复杂模块（CC=46）。并发运行多个场景，支持断点续跑（state.json），处理压力级别叠加，写出 JSONL 结果。 ### generation/ **本质**：五阶段 LLM 工厂，从 workspace 定义出发，逐步生成状态机 → 函数定义 → 政策约束 → 任务消息 → 压力消息。 - **BasePipeline**：提供日志写入基础设施 - **PipelineScenarios**：驱动前四个阶段，按 domain/workspace 并行处理（ProcessPoolExecutor） - **PipelineMessages**：驱动第五阶段，向已有场景追加压力系统消息 - **五个 ScenarioManager**：各自对应一个生成阶段，内部逻辑相似（生成 → Judge 打分 → 筛选 → 重试） - **MessagesScenarioManager**：最复杂，负责为 6 种压力 tactic × 11 强度级别 × 每场景生成压力消息（D级复杂度） - **SimilarityGraph**：用 TF-IDF + 余弦相似度去除重复场景（networkx 最大权重节点删除算法） --- ## 四、关键依赖与耦合边界 ### 硬依赖 | 依赖点 | 具体说明 | 影响范围 | |--------|---------|---------| | `data/full/` Git LFS 数据 | ~770MB，未 pull 时为 LFS 指针 | 评估无法运行 | | `inputs/pressure_categories/*.json` | 6个压力 tactic 定义文件 | 消息生成流水线 | | `inputs/workspaces/<domain>/*.json` | Workspace 角色定义 | 生成流水线入口 | | `generation_output/` 中间产物 | Step1 → Step2 的传递媒介 | messages pipeline | | POSIX `fcntl` 模块 | 跨进程限速锁 | Windows 上失效 | | `API_KEYS` 环境变量 | Client 初始化断言 | 所有 LLM 调用 | | LiteLLM | 统一 LLM 调用层 | 全系统 | ### 软依赖（可替换） -
